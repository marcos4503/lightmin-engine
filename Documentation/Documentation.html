What is Lightmin Engine Lightmin Engine (a combination of the words Light and Minimalistic) is an Engine aimed for creating Websites. It aims to provide a simple, fast, easy and powerful way to create dynamic websites, that is, websites that behave like applications! Dynamic websites are responsive websites, which often behave like applications, having a loading screen when the user accesses that website, and then the user no longer leaves that same page and only its content, as well as the interface, changes as the user navigates and uses the website, exactly how an application behaves.

The Lightmin Engine aims to be an alternative for people who do not want or cannot use much more complex Frameworks/Engines that require a greater learning curve, such as Flutter, React, Angular, among others.

The Engine is written in pure PHP, HTML, CSS and JavaScript! This means that to use this Engine and develop your website, all you will need is a server that runs PHP and a program to edit HTML, CSS, JavaScript and PHP codes! You won't need anything more than that and you'll be able to create your dynamic website in a matter of minutes after understanding the basics of this Engine!

Lightmin Engine Logo What is the dynamic behind Lightmin? Lightmin is an engine for creating dynamic websites based on Pieces, Pages and a Client. These are the three main pillars of Lightmin. The Client behaves like the base design of your dynamic website, being the base interface for your dynamic website, like mobile apps. Once the user enters your dynamic website, they will be entering the Client and will not leave there, unless they access another website or close the browser tab. The Pages work like standard HTML pages, however, they are rendered INSIDE the Client and whenever the user clicks on a link or button, and it is necessary to change to a new page, the user will not leave the Client, but instead, the current Page inside the Client will be unloaded and then the next Page will be loaded and then displayed inside the Client! You can watch the video below to understand better. In the example video below, I will show how MEGA dynamic website works. It wasn't made using Lightmin, but it works exactly like websites made with Lightmin! Using the MEGA website as an example... The Pieces act like Components in applications. Basically, Pieces are REUSABLE Components in which you use HTML, JavaScript and CSS to create them. Once you create a Piece, such as example, a Smart Text Field, you can render that Piece as many times as you want within Pages! And yes! If you decide to edit your Piece in the future, you will only need to edit the Piece file and then all existing Pieces on any Page will be updated too! The concept of Pieces makes it easier to maintain, organize and develop your websites!

Add the concepts of Client plus Pages plus Pieces plus HTML, CSS, PHP and the JavaScript that you already know and are used to, and then we have the Lightmin Engine! What do I need before start? The Lightmin Engine was developed to enable developers to create their dynamic websites (or web applications) in the most enjoyable, quick and easy way possible. It was developed so that you don't need to have compiler programs or anything similar on your PC, all you need is your standard code editor. But there are some requirements necessary for you to use Lightmin, some are optional. See below. Prior knowledge of HTML, CSS, JavaScript, PHP and how to create/host Websites. A server that runs PHP. Everyone who will access your website will need Browsers that support JavaScript. An HTML, JavaScript, CSS and PHP code editor. File structure of a Lightmin Website When downloading the Lightmin base Website from the Github repository, before editing it and making your own Website from it, you will see several files and folders. In the table below you can see what each folder and file is, as well as explanations about each one. 20%=>20%=>auto File=>Type=>Description engine=>Folder=>These are the Lightmin Engine core files, they are the files that are behind everything, making everything work. You do not need (and do not recommend) touching any of the files inside this folder. However, if in the future you want to update the Lightmin that is running behind your Website, you will only need to replace this folder with the "engine" folder of the new version of Lightmin Engine! libraries=>Folder=>Inside this folder you must place your Website CSS and JS files. I STRONGLY recommend that ALL of your Website CSS be inside this folder (except the "Pieces" CSS) so that organization is maintained, and to avoid clutter. medias=>Folder=>Inside this folder you must place ALL the media files from your Website. Be it a pure text file, image or even video. Files present here can also be easily accessed via JavaScript using the Lightmin Engine JS API! pages=>Folder=>In this folder you must place ALL the "Pages" files from your Lightmin Website. Any "Page" file that you intend to render within the Client must be present here. The "Page" files can contain JavaScript within them, without any problems, but it is recommended that they only be small scripts. Any large script should go to the "libraries" folder for greater organization of the Website. pieces=>Folder=>In this folder you must place all the XML files of the "Pieces" that you created for your Website. When downloading the base website from the Github repository, some standard "Pieces" will come so that you have at least the basics to get started! The "Pieces" must contain their JavaScript code within the XML file, regardless of the size of the script, along with the HTML and CSS code of the "Piece", for greater organization. Additionally, this makes it easier to share "Pieces" between different Lightmin websites. screens=>Folder=>In this folder, you must place all the "Screen" files for your Website. These files works as additional screens that cover your Website Client when called by JavaScript code, and this way you can use them to extend the base interface beyond just the Client interface, creating additional screens with different layouts and interfaces to perform smaller tasks on your Website, such as Login screens, Preferences screens, etc. client.php=>File=>This is your Website "Client" file, this is your Website central file. This is where you will program the layout and base interface of your Website, and it is where you will display "Pages" and other things. manifest.json=>File=>This is the Manifest of your Lightmin Website. Here in this file you must declare all the "Pieces", "Screens" and "Libraries (CSS and JS)" files present on your Lightmin Website. Only then will these files work and be compiled at runtime on the Client. settings.php=>File=>In this file you can configure any type of configurable thing for your Lightmin Website, which can range from colors of Engine elements, to transition animations between "Pages" in the Client.
